## 1. 🎯 本节内容

- markdown 标题语法

## 2. 🫧 评价

笔记中介绍了两种风格的标题语法，包括：ATX 风格和 Setext 风格。个人更加推荐的是 ATX 风格，因为它更简洁、易读，并且支持更多层级的标题。Setext 风格虽然也可以使用，但由于其限制较多，且不如 ATX 风格直观，因此在实际写作中较少使用。

## 3. 🤔 markdown 标题有哪几种写法？

markdown 标题主要有两种写法：ATX 风格和 Setext 风格。

- ATX 风格使用 `#` 号前缀，`#` 的数量表示标题层级（1-6 级）。
- Setext 风格使用在文字下方添加 `=` 或 `-` 符号的方式来定义标题，但只支持两个层级。

## 4. 🤔 ATX 风格标题和 Setext 风格标题有何区别？

- ATX 风格：使用 1-6 个 `#` 号开头，支持 6 个层级，写法简洁，是目前最常用的标题写法。
- Setext 风格：在标题文本下方用 `=` 表示一级标题，用 `-` 表示二级标题，只支持两个层级，且标题内容行的下一行至少需要一个符号 `=` 或 `-`。

::: code-group

```[ATX 风格标题]
# 一级标题

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题
```

```[Setext 风格标题]
一级标题 1
=======

一级标题 2
==

一级标题 3
=

二级标题 1
-------

二级标题 2
--

二级标题 3
-
```

## 5. 🤔 标题共有几个层级？各层级如何划分？

markdown 标题共有 6 个层级：

- 一级标题（H1）：`# 标题`，通常用于文档主标题，一篇文档一般只有一个 H1
- 二级标题（H2）：`## 标题`，用于主要章节划分
- 三级标题（H3）：`### 标题`，用于小节划分
- 四至六级：用于更细粒度的层次，实际写作中较少使用超过四级的标题

TNotes 中的笔记规范：

1. 一级标题（H1）仅用于笔记主标题，且每篇文档只能有一个 H1，位于笔记内容的开头
2. 二级标题（H2）用于主要章节划分，主要表现形式是一个个具体问句的形式
3. 三级标题（H3）用于具体问题下的小节划分，仅在必要的时候使用
4. 四至六级：很少用

## 6. 🤔 标题使用时有哪些常见注意事项？

- `#` 号后面需要加一个空格，否则部分解析器不会识别为标题
- 不建议跳级使用标题（如从 H1 直接跳到 H3），会影响文档结构的语义和可访问性
- 一篇文档通常只设置一个 H1 标题，对应文档的主标题
- 标题内容应简洁清晰，避免过长
- 标题前后建议留有空行，提高源文件可读性

## 7. 🤔 标题在文档结构中起到什么作用？

标题是文档结构的骨架。它们不仅在视觉上区分不同层次的内容，还具有以下作用：

- 自动生成目录（TOC）：许多工具可根据标题层级自动生成文档目录
- 锚点导航：每个标题会生成一个可跳转的锚点，方便文档内部链接
- SEO 优化：在转换为 HTML 发布时，标题对应 H1-H6 标签，有助于搜索引擎理解内容层次
- 辅助工具访问：屏幕阅读器依赖正确的标题层级来帮助视障用户导航文档
