## 2. 🫧 评价

在实际书写笔记时，表格用的还是蛮多的，尤其在书写一些需要突出对比的内容时。

复杂表格用的较少，往往在整个知识库中，复杂表格只会用到那么一两个，通常用于对整个知识库中的某些特定数据做一个汇总，以便快速查阅。

## 3. 🤔 如何在 markdown 中创建基本表格？

使用 `|` 分隔列，用 `-` 组成分隔行（放在表头行和内容行之间）：

```
| 姓名 | 年龄 | 城市 |
| ---- | ---- | ---- |
| 张三 | 25   | 北京 |
| 李四 | 30   | 上海 |
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-07-07-55-32.png)

## 4. 🤔 如何设置表格列的对齐方式？

在分隔行的 `-` 中添加冒号 `:` 控制对齐：

```
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 1   |   2   |   3 |
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-07-07-56-09.png)

- `:-` 或 `:---` 表示左对齐（默认）
- `:-:` 或 `:---:` 表示居中对齐
- `-:` 或 `---:` 表示右对齐

## 5. 🤔 表格中可以使用哪些 markdown 格式？

表格单元格内支持部分行内 markdown 格式，包括：链接、图片、行内代码、加粗、斜体、删除线等。不支持多行内容、列表、代码块等块级元素。

## 6. 🤔 如何在 markdown 表格中实现换行？

如果需要在单元格中换行，可以使用 HTML 的 `<br>` 标签。

示例：

```
| 姓名 | 年龄 | 城市 |
| ---- | ---- | ---- |
| 张三 | 25   | 北京<br>朝阳区 |
| 李四 | 30   | 上海<br>浦东新区 |
```

## 7. 🤔 如何创建复杂表格？（比如带有合并单元格的表格）

标准 markdown 表格语法不支持合并单元格（colspan 或 rowspan）。如有此需求，需要直接使用 HTML 的 `<table>` 标签来实现，在支持内嵌 HTML 的 markdown 渲染环境下（如大多数静态网站生成器和 GitHub）可以正常工作。

示例：

```html
<table>
  <tr>
    <!-- 合并两列 -->
    <th colspan="2">姓名</th>
    <th>年龄</th>
  </tr>
  <tr>
    <td>张三</td>
    <td>李四</td>
    <td>25</td>
  </tr>
</table>
```

## 8. 🤔 大型表格有哪些编写技巧？

- 使用编辑器的表格格式化插件（如 VS Code 的 Markdown All in One）自动对齐列宽，提高源码可读性
- 列数较多时，优先考虑是否可以拆分为多个小表格，或改用列表来呈现信息
- 使用制表符和空格对齐时，注意不同编辑器的处理方式可能有差异
- 数据量大时，建议拆分数据源到独立文件中，并利用脚本来生成 markdown 表格，而不是手动维护

## 9. 🤔 如何评价复杂表格？

我们暂且将“复杂表格”定义为：“格式复杂的表格”，比如：

- 需要合并单元格（colspan 或 rowspan）的表格
- 需要在单元格内使用多行内容的表格
- ……

markdown 对格式复杂的表格，提供的支持有限，因此也很少会用，虽然可以写 HTML 来创建复杂表格，但书写成本略高，且可读性不强。

因此，对于复杂表格，可考虑独立利用 json、yml 等格式来维护结构化的数据源，再通过脚本来解析数据源，创建 HTML table 表格。若你的 markdown 文档是基于 vitepress 这样的工具来构建的，那么你可以将解析逻辑封装成一个工具函数，并自行扩展 markdown 语法来对复杂表格数据提供解析支持。

同样的，我们也可以认为“含有大量数据的表格”是复杂表格，对于这种情况，也同样推荐拆分数据源 + 手写解析逻辑的方式来创建复杂表格。
