## 1. 🎯 本节内容

- 在 markdown 中创建分割线
- 区分分割线写法与 Setext 标题写法

## 2. 🫧 评价

TNotes 中使用的创建分割线的语法：`---`，通常用于在某个特定标题下对内容进行分组，使用频率相对较低，更多情况下优先会使用标题来控制内容的分组而非分割线。

## 3. 🤔 如何在 markdown 中创建分割线？

在单独一行写三个或更多的连字符 `-`、星号 `*` 或下划线 `_`：

```
以下写法均可生成分割线：

下面是分割线

---

下面是分割线

- - -

下面是分割线

***

下面是分割线

* * *

下面是分割线

___


下面是分割线

_ _ _
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-07-11-09-00.png)

连字符、星号、下划线各自单独使用，数量不少于三个，符号之间允许有空格，行首行尾不应有额外字符（除空格外）。不同符号混用通常不合法。

## 4. 🤔 分割线写法与 Setext 标题写法如何区分？

Setext 风格的二级标题写法 `---` 与分割线写法完全相同，解析器通过上下文来判断：如果 `---` 前面紧跟着一行非空文本，则解析为二级标题；如果前面是空行或文件开头，则解析为分割线。因此，写分割线时建议在前面留一个空行，以避免与 Setext 标题产生歧义。

```
这一行会被解析为二级标题
---

这一行是普通文本，不会被解析为二级标题。

---
```

## 5. 🤔 分割线的使用场景与规范是什么？

分割线用于在文档中创建主题上的分隔，适合在章节之间增加视觉间隔，或用于标记文档内容的逻辑过渡。不过，现代文档写作中，通常优先用标题层级来组织文档结构，分割线的使用应克制，避免过多使用导致文档结构不清晰。
