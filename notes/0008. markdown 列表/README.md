## 3. 🤔 如何在 markdown 中创建无序列表？

使用 `-`、`*` 或 `+` 加空格开头：

```
- 香蕉
- 苹果
- 橙子
```

三种符号效果相同，但同一列表内建议统一使用一种符号。

## 4. 🤔 如何在 markdown 中创建有序列表？

使用数字加英文句点和空格开头：

```
1. 第一步
2. 第二步
3. 第三步
```

有趣的是，有序列表的起始数字会决定渲染时的起始编号，但后续数字的具体值在大多数解析器中不影响输出（自动递增）。

## 5. 🤔 如何创建嵌套列表？

在子列表项前添加 2 或 4 个空格缩进：

```
- 水果
  - 香蕉
  - 苹果
- 蔬菜
  - 胡萝卜
```

无序列表和有序列表也可以互相嵌套。

## 6. 🤔 markdown 中如何创建任务列表（待办事项）？

使用 `- [ ]` 表示未完成，`- [x]` 表示已完成（GFM 扩展语法）：

```
- [x] 已完成的任务
- [ ] 未完成的任务
```

在 GitHub 等支持 GFM 的平台上，任务列表项会渲染为可交互的复选框。

## 7. 🤔 列表与段落之间的空行有什么影响？

如果列表项之间没有空行，列表项内容会被渲染为紧凑的行内文本。如果列表项之间存在空行，每个列表项的内容会被渲染为一个独立的段落（带有段落间距），视觉上会更加宽松。这个差异在大多数解析器中都会体现。此外，列表前后建议各留一个空行，以确保解析器正确识别列表边界。
