## 2. 🫧 评价

列表的语法非常简单，无非就是添加一个符号开头，然后添加列表项内容。支持无序、有序、任务列表多种列表类型。也支持通过缩进来实现嵌套列表项，实现不同层级的列表结构。

以下是关于列表部分的个人使用习惯：

- 无序列表推荐使用 `-`
- 有序列表推荐在书写的时候正确编写编号
- 无序列表用的比有序列表多
- 同一组列表项中不要有空行，一组列表项的前后应该添加空行与前后内容分隔开
- 列表的嵌套层级一般不会过深，通常 ≤ 3 层
- 书写嵌套列表的时候，同一层要么都是有序列表，要么都是无序列表，不会在同一层中同时混用有序列表和无序列表
- 任务列表的语法在常规笔记中使用频率较低，更多在特定笔记中表达某个东西是否完成的情况下使用

## 3. 🤔 如何在 markdown 中创建无序列表？

使用 `-`、`*` 或 `+` 加空格开头：

```
- 香蕉
* 苹果
+ 橙子
```

三种符号效果相同，但同一列表内建议统一使用一种符号。

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-02-23-49-23.png)

## 4. 🤔 如何在 markdown 中创建有序列表？

使用数字加英文句点和空格开头：

::: code-group

```[写法 1]
1. 第一步
2. 第二步
3. 第三步
```

```[写法 2]
1. 第一步
1. 第二步
1. 第三步
```

:::

有趣的是，有序列表的起始数字会决定渲染时的起始编号，但后续数字的具体值在大多数解析器中不影响输出（自动递增），因此上述两种写法在很多 markdown 解析器看来是等效的。

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-02-23-51-05.png)

## 5. 🤔 如何创建嵌套列表？

在子列表项前添加 2 或 4 个空格缩进：

```
- 水果
  - 香蕉
  - 苹果
- 蔬菜
  - 胡萝卜
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-02-23-53-33.png)

无序列表和有序列表也可以互相嵌套。

```
1. 水果
  - 香蕉
  - 苹果
2. 蔬菜
  - 胡萝卜
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-02-23-55-38.png)

或者：

```
- 水果
  1. 香蕉
  2. 苹果
- 蔬菜
  1. 胡萝卜
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-03-00-01-59.png)

## 6. 🤔 markdown 中如何创建任务列表（待办事项）？

使用 `- [ ]` 表示未完成，`- [x]` 表示已完成（GFM 扩展语法）：

```
- [x] 已完成的任务
- [ ] 未完成的任务
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-03-00-14-43.png)

在 GitHub 等支持 GFM 的平台上，任务列表项会渲染为可交互的复选框。

## 7. 🤔 列表与段落之间的空行有什么影响？

- 如果列表项之间没有空行，列表项内容会被渲染为紧凑的行内文本。
- 如果列表项之间存在空行，每个列表项的内容会被渲染为一个独立的段落（带有段落间距），视觉上会更加宽松。

这个差异在大多数解析器中都会体现。此外，列表前后建议各留一个空行，以确保解析器正确识别列表边界。

```
- 北京
- 上海
- 杭州

1. 苹果
2. 桃子
3. 西瓜
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-03-00-11-35.png)

中间添加一个空行，表示不同的内容，解析器在解析的时候会将没有空行的列表项紧凑渲染。
