## 3. 🤔 如何在 markdown 中插入行内代码？

使用单个反引号 `` ` `` 包裹代码内容：

```
使用 `console.log()` 打印输出。
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-05-07-54-18.png)

行内代码通常用于在段落中提及变量名、函数名、命令或短代码片段。

## 4. 🤔 如何创建多行代码块？

### 4.1. 方式一：围栏代码块（推荐）

- 使用三个反引号 ` ``` `
- 或三个波浪号 `~~~` 包裹代码

````
第一个代码块：

```
多行代码内容
第二行
```

第二个代码块：

~~~
多行代码内容
第二行
~~~
````

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-05-07-55-51.png)

### 4.2. 方式二：缩进代码块

- 每行代码前添加 4 个空格或 1 个 Tab。
- 缩进代码块不能指定语言，属于早期语法，现代写作中已较少使用。

```
代码块：

    多行代码内容
    第二行
```

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-05-07-57-40.png)

### 4.3. 两种写法对比

- 围栏代码块（3 个反引号或波浪号）是现代推荐方式，支持指定语言、不依赖缩进层级，在嵌套列表中使用更灵活。
- 缩进代码块（4 个空格 / Tab 缩进）是 markdown 最初规范中的代码块写法，语法更隐式，不支持指定语言，且在已缩进的内容（如列表项）中使用时会出现歧义，目前已逐渐被围栏代码块取代。

| 对比维度 | 围栏代码块 | 缩进代码块 |
| --- | --- | --- |
| 语法标记 | 3 个反引号 ` ``` ` 或 3 个波浪号 `~~~` | 每行前 4 个空格或 1 个 Tab |
| 推荐使用 | ✅ 现代推荐方式 | ❌ 已逐渐被取代 |
| 指定语言 | ✅ 支持语法高亮 | ❌ 不支持 |
| 缩进依赖 | ✅ 不依赖缩进层级 | ✅ 依赖缩进 |
| 嵌套列表 | ✅ 使用更灵活 | ❌ 容易出现歧义 |
| 语法特点 | 显式标记 | 隐式语法 |
| 规范版本 | 现代规范 | Markdown 最初规范 |

## 5. 🤔 代码块如何指定编程语言实现语法高亮？

在围栏代码块的开头反引号后紧跟语言标识符：

````
```javascript
const greet = (name) => `Hello, ${name}!`;
```
````

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-05-08-02-45.png)

常见语言标识符：`javascript`、`python`、`bash`、`html`、`css`、`json`、`sql` 等。

是否真正渲染语法高亮取决于渲染环境是否加载了高亮库（如 highlight.js、Prism.js 等）。

## 6. 🤔 如何在代码块中显示反引号字符？

行内代码：如果代码内容本身包含反引号，可以用多个反引号作为分隔符：

````
在行内书写反引号：

`` 包含`反引号`的文本 ``

多个反引号：` ``` `

在代码块内书写反引号：

```
这是单个反引号 `
这是多个连续的反引号 ```
```
````

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs-2026@main/2026-03-05-08-15-07.png)

- 如果围栏代码块自身需要用反引号作为结束标记但内容中也有三个反引号，可以使用 4 个反引号或单个反引号作为围栏。
- 代码块内部的内容不会被解析为 markdown，因此反引号可以直接写入代码块中，无须特殊处理。
