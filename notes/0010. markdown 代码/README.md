## 3. 🤔 如何在 markdown 中插入行内代码？

使用单个反引号 `` ` `` 包裹代码内容：

```
使用 `console.log()` 打印输出。
```

行内代码通常用于在段落中提及变量名、函数名、命令或短代码片段。

## 4. 🤔 如何创建多行代码块？

方式一，围栏代码块（推荐）：使用三个反引号 ` ``` ` 或三个波浪号 `~~~` 包裹代码。

````
```
多行代码内容
第二行
```
````

方式二，缩进代码块：每行代码前添加 4 个空格或 1 个 Tab。缩进代码块不能指定语言，属于早期语法，现代写作中已较少使用。

## 5. 🤔 代码块如何指定编程语言实现语法高亮？

在围栏代码块的开头反引号后紧跟语言标识符：

````
```javascript
const greet = (name) => `Hello, ${name}!`;
```
````

常见语言标识符：`javascript`、`python`、`bash`、`html`、`css`、`json`、`sql` 等。是否真正渲染语法高亮取决于渲染环境是否加载了高亮库（如 highlight.js、Prism.js 等）。

## 6. 🤔 围栏代码块和缩进代码块有何区别？

围栏代码块（3 个反引号或波浪号）是现代推荐方式，支持指定语言、不依赖缩进层级，在嵌套列表中使用更灵活。缩进代码块（4 个空格 / Tab 缩进）是 markdown 最初规范中的代码块写法，语法更隐式，不支持指定语言，且在已缩进的内容（如列表项）中使用时会出现歧义，目前已逐渐被围栏代码块取代。

## 7. 🤔 如何在代码块中显示反引号字符？

行内代码：如果代码内容本身包含反引号，可以用多个反引号作为分隔符：

```
`` 包含`反引号`的文本 ``
```

代码块：代码块内部的内容不会被解析为 markdown，因此反引号可以直接写入代码块中，无须特殊处理。如果围栏代码块自身需要用反引号作为结束标记但内容中也有三个反引号，可以使用 4 个反引号作为围栏。
